Object-Oriented Programming (OOP) is a programming paradigm that organizes software around **objects** rather than functions. An object combines **data (variables)** and **behavior (methods)** into a single unit.

### Why OOP?

OOP makes code:

* Easier to understand
* Reusable
* Secure
* Easier to maintain and extend
* Suitable for large applications

# 1. Class

A **class** is a blueprint or template for creating objects. It defines what data an object will have and what actions it can perform.

### Example

```java
class Student {

    String name;
    int age;

    void display() {
        System.out.println(name + " " + age);
    }

}
```

Here:

* `Student` → Class
* `name`, `age` → Variables (Attributes)
* `display()` → Method (Behavior)

Think of a class like a **house blueprint**. The blueprint itself is not a house—it just describes how houses should be built.

# 2. Object

An **object** is an instance of a class. It occupies memory and can use the variables and methods defined in the class.

### Example

```java
public class Main {

    public static void main(String[] args) {

        Student s1 = new Student();

        s1.name = "Jay";
        s1.age = 25;

        s1.display();

    }

}
```

Output

```
Jay 25
```

Memory representation:

```
Student Class
-------------------
name
age
display()

        ↓

Object s1
-------------------
name = Jay
age = 25
```

You can create multiple objects from the same class:

```java
Student s1 = new Student();
Student s2 = new Student();
Student s3 = new Student();
```

Each object has its own data.

# 3. Encapsulation

Encapsulation means **wrapping data and methods into a single unit** and **hiding sensitive data** from direct access.

Instead of allowing anyone to change variables directly, we use **private** variables and provide controlled access through getters and setters.

### Without Encapsulation

```java
class Student {

    public int age;

}
```

Anyone can modify the age:

```java
Student s = new Student();

s.age = -100;
```

This is invalid but allowed.

---

### With Encapsulation
IN Encapsulation, we need to set (Getter and setter to access the private defined variable for class, we directly can't access it without getter and setter - so we can place a validation to getter and setter to define a criteria like age should be greater than 0 etc)  
```java
class Student {

    private int age;

    public void setAge(int age) {

        if(age > 0)
            this.age = age;

    }

    public int getAge() {
        return age;
    }

}
```

Usage

```java
Student s = new Student();

s.setAge(25);

System.out.println(s.getAge());
```

Output

```
25
```

Invalid input:

```java
s.setAge(-20);
```

The setter can reject invalid values.

### Advantages

* Data hiding
* Better security
* Validation
* Easier maintenance

Real-life example:

Think of an **ATM**.

```
User
   ↓
ATM Machine
   ↓
Bank Database
```

You cannot directly access the bank database. You interact through the ATM interface.

# 4. Inheritance

Inheritance allows one class to **inherit properties and methods** from another class.

### Parent Class

```java
class Animal {

    void sound() {

        System.out.println("Animal makes sound");

    }

}
```

### Child Class

```java
class Dog extends Animal {

    void bark() {

        System.out.println("Dog barks");

    }

}
```

Usage

```java
public class Main {

    public static void main(String[] args) {

        Dog d = new Dog();

        d.sound();

        d.bark();

    }

}
```

Output

```
Animal makes sound
Dog barks
```

Diagram

```
Animal
   ↑
  Dog
```

Dog automatically gets everything from Animal.

### Types of Inheritance in Java

#### Single

```
Animal
   ↑
 Dog
```

#### Multilevel

```
Animal
   ↑
 Dog
   ↑
 Puppy
```

#### Hierarchical

```
      Animal
      /    \
    Dog    Cat
```

**Java does not support multiple inheritance with classes** because it can create ambiguity (the "diamond problem"). It supports multiple inheritance through interfaces.

Advantages:

* Code reuse
* Less duplication
* Easier maintenance

# 5. Polymorphism

Polymorphism means **one interface, many forms**.

The same method name can behave differently depending on the context.

There are two types.

## A. Compile-Time Polymorphism (Method Overloading)

Same method name, different parameters.

```java
class Calculator {

    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }

}
```

Usage

```java
Calculator c = new Calculator();

System.out.println(c.add(5, 10));

System.out.println(c.add(5, 10, 15));
```

Output

```
15
30
```

---

## B. Runtime Polymorphism (Method Overriding)

Parent class

```java
class Animal {

    void sound() {
        System.out.println("Animal sound");
    }

}
```

Child class

```java
class Dog extends Animal {

    @Override
    void sound() {
        System.out.println("Dog barks");
    }

}
```

Usage

```java
Animal a = new Dog();

a.sound();
```

Output

```
Dog barks
```

Although the reference type is `Animal`, the method of `Dog` is called. This decision is made at runtime.

Real-life example:

A remote control has a **Power** button. Pressing it turns on a TV, an AC, or a projector. The button is the same, but the behavior depends on the device.

# 6. Abstraction

Abstraction means **showing only the essential details while hiding the implementation**.

For example, when you drive a car, you use the steering wheel, pedals, and gear lever. You don't need to know how the engine works internally.

Java supports abstraction using **abstract classes** and **interfaces**.

## Abstract Class

```java
abstract class Animal {

    abstract void sound();

}
```

Child class

```java
class Dog extends Animal {

    @Override
    void sound() {
        System.out.println("Dog barks");
    }

}
```

Usage

```java
Animal a = new Dog();

a.sound();
```

Output

```
Dog barks
```

You cannot create an object of an abstract class:

```java
Animal a = new Animal();   // Error
```

## Interface Example

```java
interface Vehicle {

    void start();

}
```

Implementation

```java
class Car implements Vehicle {

    @Override
    public void start() {
        System.out.println("Car Started");
    }

}
```

Usage

```java
Vehicle v = new Car();

v.start();
```

Output

```
Car Started
```

# Summary

| Concept       | Definition                                          | Example                               |
| ------------- | --------------------------------------------------- | ------------------------------------- |
| Class         | Blueprint for objects                               | `class Student {}`                    |
| Object        | Instance of a class                                 | `new Student()`                       |
| Encapsulation | Hide data and provide controlled access             | `private` fields with getters/setters |
| Inheritance   | Child class inherits from parent class              | `class Dog extends Animal`            |
| Polymorphism  | Same interface, different behavior                  | Method overloading and overriding     |
| Abstraction   | Hide implementation details, expose only essentials | Abstract classes and interfaces       |

### How the concepts work together

Imagine an online shopping application:

* **Class:** `Product`, `Customer`, and `Order` define the structure of the application.
* **Object:** `Product laptop = new Product();` represents a specific laptop.
* **Encapsulation:** Product price is kept `private` and updated only through validated methods.
* **Inheritance:** `CreditCardPayment` and `UPIPayment` inherit from a common `Payment` class.
* **Polymorphism:** Calling `payment.pay()` executes different payment logic depending on the payment type.
* **Abstraction:** The checkout process only knows it needs to call `pay()`, without needing to know how each payment method is implemented.
